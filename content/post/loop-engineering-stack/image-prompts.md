# 文章配图生成 Prompt

生成后将图片保存到本目录，格式为 .webp 或 .png（会自动转为 .webp），文件名与文章中引用一致。

统一视觉要求：Ghost Protocol 深色科技风，深海军蓝背景 `#07090f/#0d1117`，主强调色青绿 `#2dd4bf`，辅助色琥珀 `#f59e0b`，少量紫色和红色高亮；中文信息图风格，干净、锐利、适合技术博客；不要真实品牌 Logo，不要英文字母乱码；画面留出足够呼吸空间，9:16 竖版，至少 1080x1920。

---

## 1. cover.webp — 文章封面

一张中文技术博客封面信息图，主题是“Loop Engineering：Agent 不是跑一次，而是活在循环里”。画面中心是一个发光的四层同心循环系统，内层是一个 AI agent 核心节点，外层依次是“执行循环”“验证循环”“事件循环”“爬坡循环”；循环之间有流动箭头和状态轨迹。背景是深色开发者工作台、trace 日志、工具调用节点的抽象线框。右下角可以有一个极简的“鬼哥”极客剪影形象：戴眼镜、技术博主气质、站在控制台前观察循环系统。中文标题清晰可读：`Loop Engineering`、副标题：`Agent 不是跑一次，而是活在循环里`。Ghost Protocol 深色科技风，青绿和琥珀高亮，9:16。

---

## 2. loop-stack.webp — 四层 Loop Engineering Stack

生成一张中文信息图，展示 Agent 系统的四层 loop stack。画面采用纵向分层架构，从下到上依次为：`Agent Loop：观察-决策-行动-更新状态`、`Verification Loop：测试-裁判-反馈-修正`、`Event-driven Loop：事件唤醒-恢复上下文-挂起等待`、`Hill Climbing Loop：trace-eval-feedback-系统改进`。每层使用不同但协调的线框模块和箭头，层与层之间有清晰的数据流。左侧放小图标：机器人、盾牌检查、闪电事件、山峰上升。中文标签必须清楚准确。深色背景，青绿为主，琥珀用于关键节点，少量紫色表示反馈，9:16。

---

## 3. verification-loop.webp — 独立验证循环

一张解释“独立 verification loop”的中文流程图。画面中间是一个闭环：`Agent 提交改动` -> `确定性检查：测试 / lint / schema` -> `独立 LLM Judge` -> `通过继续 / 失败反馈` -> 回到 Agent。强调“裁判独立于生成上下文”。可以用两个分离的发光面板表现：左侧是 Agent 工作区，右侧是 Judge 审查室，中间通过结构化反馈连接。不要出现真实公司 Logo。整体有工程仪表盘感，包含测试通过/失败的小状态灯。深色科技风，青绿、琥珀、红色错误高亮，中文可读，9:16。

---

## 4. event-driven-agent.webp — 事件驱动 Agent

一张中文信息图，主题是“Event-driven Agent：从聊天机器人到后台工作人员”。画面中一个 AI agent 处于中央控制台，周围有多个事件源向它发送信号：`用户消息`、`GitHub Issue`、`CI 失败`、`数据库异常`、`定时任务`、`人工审批`。Agent 的处理链路显示为：`加载状态` -> `决定动作` -> `调用工具 / 请求人类` -> `持久化 trace` -> `等待下一事件`。视觉上像一个实时事件总线和任务编排系统，不要卡通化。深色背景，细线网络，青绿主色，琥珀事件点，9:16。

---

## 5. hill-climbing-loop.webp — 从失败到复利的爬坡循环

一张中文信息图，主题是“Hill Climbing Loop：失败如何变成系统复利”。画面是一座抽象的山，Agent 系统沿着山坡迭代上升，每一步都有 trace 轨迹点。山坡旁边展示闭环：`traces`、`evals`、`human feedback`、`prompt / policy changes`、`regression checks`。底部是混乱失败案例，顶部是更稳定的生产 Agent。画面要体现“越跑越好”的长期优化，不要像商业营销海报，要像技术架构图和研究笔记的结合。中文标签清晰，深色科技风，青绿主色，琥珀表示改进收益，少量紫色表示人类反馈，9:16。

---

## 使用说明

1. 将上述 prompt 分别输入 AI 图片生成工具（如 Midjourney, DALL-E, Ideogram 等）
2. 建议使用 9:16 比例，分辨率至少 1080x1920
3. 生成后保存到本目录（PNG 或 WebP 均可，后续会统一转为 WebP）
4. 文章中已经用 `![描述](文件名.webp)` 格式引用了这些图片
