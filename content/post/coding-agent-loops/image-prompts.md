# 文章配图生成 Prompt

文章：`别再只会提示词了：下一代程序员要会设计 Agent 循环`

用途：为本文生成 5 张中文技术博客配图。生成后保存到本目录，文件名必须与文章引用一致：

- `cover.webp`
- `prompt-to-loop.webp`
- `automation-heartbeat.webp`
- `maker-checker.webp`
- `coding-agent-loop.webp`

参考角色图：`~/Downloads/guige.png`

使用方式：把 `~/Downloads/guige.png` 作为“鬼哥角色参考图”，不是编辑底图。生成图里可以出现一个小型鬼哥旁白/监督角色，但不要照搬参考图里的真实品牌 Logo、电脑 Logo、黑色遮挡块或咖啡馆背景。

## 统一视觉规范

- 语言：中文为主，保留必要英文技术词，如 `Prompt`、`Agent Loop`、`CI`、`Worktree`、`Sub-agent`、`Connector`、`state.md`。
- 画幅：9:16 竖版，至少 1080×1920。
- 风格：中文技术博客 infographic，深色科技风，清晰、锐利、结构化，适合 AI 工程主题。
- 视觉基调：Ghost Protocol / dark engineering dashboard / production system map。
- 背景：深海军蓝或近黑色，建议 `#07090f` / `#0d1117`。
- 主强调色：青绿 `#2dd4bf`。
- 辅助强调色：琥珀 `#f59e0b`。
- 少量高亮：紫色表示 sub-agent / 工具链，红色表示失败反馈或风险。
- 文字：所有中文标签必须清楚、准确、可读；不要生成乱码、错别字、伪中文。
- 禁止：真实品牌 Logo、水印、二维码、随机英文装饰字、过度卡通化、营销海报感、把鬼哥角色放得比信息结构更显眼。
- 鬼哥角色：Q 版年轻男性，棕色凌乱头发，橙色头带写 `鬼哥`，蓝色 hoodie，专注或轻微吐槽表情。角色只作为小型旁白、监督员、指示牌或角落 sticker。

## 1. `cover.webp` — 文章封面

生成一张中文技术博客封面信息图。

主题：`别再只会提示词了：下一代程序员要会设计 Agent 循环`

核心画面：中心是一个发光的 `Agent Loop` 控制台。人类工程师不再逐句提示机器人，而是在设计一套自动循环系统。画面要体现“从提示模型，到设计系统”的迁移。

必须出现的文字：

- 主标题：`Agent 循环设计`
- 副标题：`从提示模型到设计系统`
- 循环节点：`发现任务`、`分配执行`、`隔离改动`、`独立验证`、`记录状态`、`决定下一步`

辅助视觉元素：

- 深色代码仓库背景
- CI 状态灯
- 任务 inbox
- Git worktree 分支线
- 外部记忆或状态面板
- 小型鬼哥监督角色，站在控制台旁边或右下角，像系统设计旁白

构图建议：中央圆环或闭环控制台占主视觉，标题在上方，底部留出工程系统细节。整体严肃、清晰，不要做成普通励志封面。

## 2. `prompt-to-loop.webp` — 从 Prompt 到 Loop

生成一张中文对比信息图，解释文章第一部分：`Prompt engineering 还没死，但它不再是杠杆最大的位置`。

核心画面：左右对比。

左侧旧模式：人类一直坐在驾驶位，逐轮给 agent 写 prompt、读结果、补上下文、再追问。视觉上略显拥挤、手动、反复。

右侧新模式：系统自己按 loop 运行，有明确的任务发现、执行、验证、记忆和下一轮机制；人类只在监督点介入。视觉上更清晰、更结构化。

必须出现的文字：

- 标题：`从 Prompt 到 Loop`
- 左侧标题：`旧模式：逐轮提示`
- 左侧流程：`人写 prompt` -> `agent 生成` -> `人读结果` -> `补充上下文` -> `继续追问`
- 右侧标题：`新模式：设计循环`
- 右侧流程：`发现任务` -> `执行` -> `验证` -> `记忆` -> `下一轮`
- 中央强调句：`Prompt 还在，但被放进更大的系统里`

辅助视觉元素：

- 旧模式可以有聊天气泡、方向盘、手动操作面板
- 新模式可以有闭环箭头、状态记录、自动触发节点
- 小型鬼哥角色指向中央强调句，表情像在提醒“别只会 prompt”

构图建议：竖版上下或左右分栏都可以，但必须明显对比“人当调度器”和“人设计系统”。

## 3. `automation-heartbeat.webp` — Automation 心跳

生成一张中文信息图，解释文章部分：`Automations 是心跳：它让循环真的会“再来一次”`。

核心画面：一个每天定时跳动的系统心脏或节拍器，连接到多个自动发现任务。它不是浪漫心脏，而是开发者系统里的自动化心跳。

必须出现的文字：

- 标题：`Automation 心跳`
- 副标题：`Automations 是 Agent Loop 的心跳`
- 中央标签：`定时触发`
- 任务节点：`Issue Triage`、`CI 失败总结`、`Commit Briefing`、`Bug Hunt`、`Skill 调用`
- 右侧收集区：`Triage Inbox`
- 关键句：`任务发现被系统化了`

辅助视觉元素：

- 定时器、日历、节拍线或心跳线
- 自动任务编排面板
- CI 红绿灯
- issue 列表、commit 摘要、bug 搜索雷达
- 需要人类处理的事项进入 `Triage Inbox`
- 小型鬼哥角色像值班工程师，在 inbox 旁边看板

构图建议：中央心跳装置向外发散任务节点，最后汇入右侧或底部 inbox。琥珀色表示定时触发，青绿色表示自动流转，红色表示 CI 失败或 bug。

## 4. `maker-checker.webp` — Maker / Checker 分离

生成一张中文流程信息图，解释文章部分：`Sub-agents：不要让写代码的人当唯一裁判`。

核心画面：一个 coding loop 里的 sub-agent 分工。重点是 Verifier 要明显独立于 Implementer，像独立审查室，不能看起来只是同一个流程里的下一步装饰。

必须出现的文字：

- 标题：`Maker / Checker 分离`
- 阶段 1：`Explorer：读上下文`
- 阶段 2：`Implementer：改代码`
- 阶段 3：`Verifier：按 spec / tests / skills 挑刺`
- 硬性检查：`tests`、`lint`、`type check`、`schema validation`
- 模糊质量审查：`架构`、`文案`、`交互`、`安全边界`
- 强调句：`循环越自动化，验证越不能省`

辅助视觉元素：

- Explorer 在读 repo、issue、spec
- Implementer 在改代码
- Verifier 坐在独立审查室，拿 checklist 或红笔挑刺
- 绿色通过路径
- 红色失败反馈回流到 Implementer
- 小型鬼哥角色站在 Verifier 旁边，像“独立挑刺”监督员

构图建议：三列流程图或阶梯流程图。底部放硬性检查条和模糊质量审查条。Verifier 的边界要明显，避免让人误解成自己审自己。

## 5. `coding-agent-loop.webp` — 一个完整 Coding Agent Loop

生成一张中文架构信息图，解释文章部分：`一个真实 loop 可以长什么样`。

核心画面：展示一个每天早晨自动运行的 coding agent loop。整体像生产级 agent 操作系统，不要卡通化，不要像玩具流程图。

必须出现的文字：

- 标题：`完整 Coding Agent Loop`
- 中心脊柱：`state.md`
- 脊柱说明：`外部记忆`
- 流程节点：
  - `9:00 Automation 启动`
  - `Triage Skill 读取 CI / Issues / Commits`
  - `写入 state.md / Linear`
  - `创建 Worktree`
  - `Sub-agent A 起草修复`
  - `Sub-agent B 验证`
  - `Connector 创建 PR / 更新 Ticket / 通知频道`
  - `未处理事项回到 Inbox`
- 人类控制点：`人工接管点`

辅助视觉元素：

- 中央 `state.md` 作为贯穿全图的外部记忆脊柱
- 周围节点组成闭环
- worktree 分支、PR、ticket、通知频道、CI 信号
- 自动化从早晨 9:00 触发
- 未处理事项回流到 inbox
- 小型鬼哥角色在侧边做系统监督员，不能遮挡流程

构图建议：竖版架构图。中心是 `state.md` 记忆脊柱，流程节点围绕它形成闭环。箭头必须清晰，不要压住文字。长标签可以拆行，但不能删改含义。

## 生成检查清单

每张图生成后检查：

1. 文件名是否与文章引用完全一致。
2. 是否为 9:16 竖图。
3. 中文标题和标签是否清楚、准确、无乱码。
4. 是否包含鬼哥角色参考特征：橙色 `鬼哥` 头带、蓝色 hoodie、专注/吐槽表情。
5. 是否没有真实品牌 Logo、水印、二维码、黑色遮挡块。
6. 信息结构是否优先于装饰，不能让角色喧宾夺主。
7. 文章里对应段落读到图片时，图片是否能解释该段核心观点。
